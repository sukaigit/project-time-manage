# 角色管理

## Purpose

维护系统中的角色定义，每个角色关联一组菜单和按钮操作权限，支持多角色权限累加。属于 RBAC 权限模型的核心部分。

## Requirements

### Requirement: 角色 CRUD

系统管理员 SHALL 可以新增、编辑、查询和删除角色。

**ID:** REQ-ROLE-001

#### Scenario: 新建角色

- 系统管理员点击「新建角色」按钮，弹出新建表单
- 表单字段包括：角色名称（必填）、角色编码（必填，唯一）、状态（默认启用）、备注（可选）
- 保存后角色出现在角色列表中

#### Scenario: 编辑角色

- 点击列表中的「编辑」按钮，弹出编辑表单
- 角色编码为只读不可修改
- 可修改：角色名称、状态、备注
- 保存后列表实时更新

#### Scenario: 删除角色

- 点击「删除」按钮前弹出二次确认弹窗
- 如该角色已关联用户，提示「该角色下有关联用户，请先解绑用户后再删除」
- 确认后删除角色

#### Scenario: 查询角色

- 查询条件：角色名称（模糊搜索）、状态（下拉选择：全部/启用/停用）
- 点击「查询」后表格刷新显示匹配结果

### Requirement: 分配权限

系统管理员 SHALL 可以为每个角色分配菜单和按钮操作权限，支持父子联动——勾选父级自动勾选子级，取消所有子级自动取消父级。

**ID:** REQ-ROLE-002

#### Scenario: 打开权限分配弹窗

- 点击角色列表中的「分配权限」按钮，弹出权限树弹窗
- 弹窗标题显示当前角色名称（如「分配权限 — 系统管理员」）

#### Scenario: 权限树

- 以树形结构展示所有菜单模块及其操作权限
- 每个菜单模块为一个分组，下级为可勾选的操作权限
- 勾选父级（菜单模块）时自动勾选所有子级操作
- 取消某个子级勾选时父级变为半选状态
- 取消所有子级时父级自动取消
- 权限列表：

| 菜单模块 | 操作权限 |
|:---------|:---------|
| 首页仪表盘 | 查看 |
| 项目管理 | 查看、新建、编辑、删除 |
| 任务管理 | 查看、新建、编辑、删除 |
| 工时管理 | 查看、新建 |
| 工时审批 | 查看、审批通过、审批驳回 |
| 员工工时统计 | 查看、导出Excel |
| 项目工时统计 | 查看、导出Excel |
| 用户管理 | 查看、新建、编辑、删除 |
| 角色管理 | 查看、新建、编辑、删除、分配权限 |

#### Scenario: 保存权限

- 点击「保存」按钮时保存所选权限
- 保存后 Toast 提示「权限分配成功」
- 下次该角色用户的权限实时生效

## Non-Functional Requirements

- 角色编码全局唯一，不可重复
- 删除角色时校验用户关联，有关联则禁止删除
