# 项目工时管理系统 — 测试案例

> 生成日期：2026-07-13

## 1. 认证模块（Auth）

### TC-AUTH-001：登录成功
- **前置条件：** 数据库已初始化，admin 用户存在
- **步骤：**
  1. 访问登录页，获取验证码
  2. 输入用户名 `admin`、密码 `uu888888`、验证码
  3. 点击登录
- **预期结果：** 登录成功，跳转首页仪表盘

### TC-AUTH-002：登录失败 — 验证码错误
- **步骤：** 输入正确用户名密码，输入错误验证码
- **预期结果：** 提示「验证码错误」

### TC-AUTH-003：登录失败 — 密码错误
- **步骤：** 输入正确用户名，错误密码，正确验证码
- **预期结果：** 提示「用户名或密码错误」

### TC-AUTH-004：退出登录
- **步骤：** 登录后 → 右上角下拉 → 点击「退出系统」
- **预期结果：** 跳转登录页，session 清除

### TC-AUTH-005：修改密码
- **步骤：** 登录后 → 右上角下拉 → 修改密码 → 输入新旧密码 → 保存
- **预期结果：** 提示修改成功

## 2. 仪表盘（Dashboard）

### TC-DASH-001：首页仪表盘展示
- **前置条件：** 以 admin 登录
- **步骤：** 登录后自动进入首页
- **预期结果：**
  - 导航栏显示「项目工时管理系统」+ 用户名 + 角色
  - 左侧菜单栏根据角色显示功能菜单
  - 统计卡片显示项目总数、任务总数、成员总数、本月工时、累计工时
  - 近期工时动态表格显示最新记录
  - TOP5 排名显示项目/员工月度及年度排名

## 3. 项目管理（Project Management）

### TC-PROJ-001：项目列表
- **步骤：** 点击左侧「项目管理」
- **预期结果：** 显示项目列表，含项目名称/编号/所属部门/成员数/状态/日期/操作列

### TC-PROJ-002：新建项目
- **步骤：** 点击「+ 新建项目」→ 填写表单 → 保存
- **预期结果：** 项目创建成功，列表刷新显示新项目

### TC-PROJ-003：编辑项目
- **步骤：** 点击项目行「编辑」→ 修改信息 → 保存
- **预期结果：** 信息更新成功，列表刷新

### TC-PROJ-004：删除项目
- **步骤：** 点击「删除」→ 确认弹窗 → 确定
- **预期结果：** 项目删除成功（如有未完成工时则不可删除）

### TC-PROJ-005：成员管理
- **步骤：** 点击「成员管理」→ 弹窗展示用户列表 → 勾选/取消 → 保存
- **预期结果：** 成员列表更新

### TC-PROJ-006：查询筛选
- **步骤：** 输入项目名称/编号 → 选择状态 → 点击查询
- **预期结果：** 表格显示匹配结果

## 5. 工时管理（Time Entry）

### TC-TE-001：工时列表
- **步骤：** 点击「工时管理」
- **预期结果：** 显示工时记录列表，含员工/项目/任务/日期/时长/状态/操作列

### TC-TE-002：新建工时（普通员工）
- **步骤：** 点击「+ 新建工时」→ 选项目（任务自动过滤）→ 填日期/时长/内容 → 保存
- **预期结果：** 工时提交成功，状态为「待审批」

### TC-TE-003：编辑已驳回工时
- **步骤：** 找到已驳回记录 → 点击「编辑」→ 修改 → 保存
- **预期结果：** 重新提交成功

### TC-TE-004：删除工时
- **步骤：** 点击「删除」→ 确认
- **预期结果：** 工时记录删除

### TC-TE-005：查询筛选
- **步骤：** 按员工/项目/日期范围/状态筛选
- **预期结果：** 表格显示匹配结果

## 6. 工时审批（Approval）

### TC-APPR-001：待审批列表
- **步骤：** 点击「工时审批」
- **预期结果：** 显示待审批的工时记录列表

### TC-APPR-002：审批通过
- **步骤：** 点击「通过」
- **预期结果：** 状态变为「已通过」

### TC-APPR-003：驳回（含原因）
- **步骤：** 点击「驳回」→ 填写驳回原因 → 确定
- **预期结果：** 状态变为「已驳回」，员工可编辑重提

### TC-APPR-004：按角色过滤
- **步骤：** 分别以 admin/部门经理/项目经理 登录查看审批页
- **预期结果：** 各角色看到对应权限范围的待审批数据
