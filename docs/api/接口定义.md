# API 接口定义

通用说明：
- 基础路径：`/api`
- 请求/响应格式：`application/json`
- 认证方式：Session（登录后自动携带）
- 权限校验：按角色 intercept

---

## 1. 认证模块

### POST /api/auth/login
登录

**请求体：**
| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----|:-----|
| username | String | 是 | 用户名 |
| password | String | 是 | 密码 |
| captcha | String | 是 | 图形验证码 |

**响应：**
```json
{ "code": 200, "data": { "token": "sessionId", "user": { "id": 1, "username": "admin", "name": "张经理", "role": "admin" } } }
```

### POST /api/auth/logout
退出

### GET /api/auth/captcha
获取验证码

**响应：** 图片字节流 + `Set-Cookie: captcha=xxxx`

### PUT /api/auth/password
修改密码

**请求体：**
| 字段 | 类型 | 必填 | 说明 |
|:-----|:-----|:----|:-----|
| oldPassword | String | 是 | 当前密码 |
| newPassword | String | 是 | 新密码（8位+字母数字） |

## 2. 项目管理

### GET /api/projects
项目列表（分页+筛选）

**参数：** page, size, name(模糊), code(模糊), status

**响应：**
```json
{ "code": 200, "data": { "total": 10, "list": [{ "id": 1, "name": "ERP", "code": "PRJ-001", "dept": "研发与交付中心", "memberCount": 12, "status": "启用", "startDate": "2026-06-01", "endDate": "2026-12-31" }] } }
```

### POST /api/projects
新建项目

**请求体：** name(必填), code(必填), dept(默认), description, status, startDate, endDate

### PUT /api/projects/{id}
编辑项目

### DELETE /api/projects/{id}
删除项目（有未完成工时的不可删）

### GET /api/projects/{id}/members
获取项目成员列表

### PUT /api/projects/{id}/members
更新项目成员

**请求体：** `{ "memberIds": [1, 2, 3] }`

## 3. 任务管理

### GET /api/tasks
任务列表（分页+筛选）

**参数：** page, size, name, projectId, status

### POST /api/tasks
新建任务

**请求体：** name(必填), code(必填), projectId(必填), status

### PUT /api/tasks/{id}
编辑任务

### DELETE /api/tasks/{id}
删除任务

---

## 4. 工时管理

### GET /api/time-entries
工时列表（分页+筛选）

**参数：** page, size, employeeId, projectId, taskId, startDate, endDate, status

### POST /api/time-entries
新建工时

**请求体：** employeeId(必填,默认当前用户), projectId(必填), taskId(项目经理可不填), date(必填), hours(必填,0.5~24), content

### PUT /api/time-entries/{id}
编辑工时（仅已驳回状态可编辑）

### DELETE /api/time-entries/{id}
删除工时

---

## 5. 工时审批

### GET /api/approvals
待审批列表（自动按角色过滤）

**参数：** page, size, projectId, employeeId, startDate, endDate, status, taskId

### POST /api/approvals/{id}/approve
审批通过

### POST /api/approvals/{id}/reject
驳回

**请求体：** `{ "reason": "必填驳回原因" }`

## 6. 工时统计

### GET /api/statistics/employees
员工工时统计

**参数：** year, month, name(模糊)

**响应：**
```json
{ "code": 200, "data": [{ "name": "张经理", "role": "项目经理", "monthHours": 168, "yearHours": 720 }] }
```

### GET /api/statistics/projects
项目工时统计

**参数：** year, month, name(模糊)

**响应：**
```json
{ "code": 200, "data": [{ "name": "ERP 系统开发", "leader": "张经理", "dept": "研发与交付中心", "monthHours": 240, "yearHours": 957 }] }
```

### GET /api/statistics/dashboard
首页仪表盘数据

**响应：** projectCount, taskCount, userCount, monthHours, yearHours, projectTop5(month+year), employeeTop5(month+year), recentActivity

---

## 7. 用户管理

### GET /api/users
用户列表（分页+筛选）

**参数：** page, size, username, name, roleId, status

### POST /api/users
新建用户（默认密码 uu888888）

**请求体：** username(必填), name(必填), roleIds(数组), dept(默认), status

### PUT /api/users/{id}
编辑用户

### DELETE /api/users/{id}
删除用户

### POST /api/users/{id}/reset-password
重置密码（重置为 uu888888）

---

## 8. 角色管理

### GET /api/roles
角色列表（分页+筛选）

### POST /api/roles
新建角色

**请求体：** name(必填), code(必填), status, note

### PUT /api/roles/{id}
编辑角色

### DELETE /api/roles/{id}
删除角色（有关联用户则禁止）

### GET /api/roles/{id}/permissions
获取角色权限

### PUT /api/roles/{id}/permissions
保存角色权限

**请求体：**
```json
{ "permissions": { "dashboard": ["查看"], "project": ["查看","新建","编辑","删除"], ... } }
```

---

### GET /api/dashboard
首页仪表盘数据（同 statistics/dashboard）

## 错误码

| 状态码 | 说明 |
|:------|:-----|
| 200 | 成功 |
| 400 | 参数错误 |
| 401 | 未登录/会话超时 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 冲突（如删除有关联的记录） |
| 500 | 服务器错误 |
